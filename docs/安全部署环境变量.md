# 安全部署环境变量

生产环境不再从仓库读取数据库密码、Redis 密码和 JWT 密钥。请在服务器上通过 systemd 的 `EnvironmentFile` 注入，配置文件权限设为仅 root 可读。

本地开发未配置 `JWT_SECRET` 时，应用会为当前进程生成临时随机密钥，因此可以正常启动，但重启后旧 token 会失效。生产环境必须显式配置固定且足够长的 `JWT_SECRET`。

## 1. 创建环境变量文件

创建 `/etc/xinke/xinke.env`：

```text
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_DATABASE=xinkeerp
MYSQL_USERNAME=xinke_app
MYSQL_PASSWORD=替换为数据库专用账号密码
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_DATABASE=0
REDIS_PASSWORD=替换为Redis密码
JWT_SECRET=替换为至少64位的随机密钥
LIVE_PLUGIN_REPORT_KEY=替换为插件上报专用随机密钥
CORS_ALLOWED_ORIGIN_PATTERNS=https://你的正式域名
FRAME_ANCESTORS=http://39.96.157.230
SWAGGER_ENABLED=false
DRUID_STAT_ENABLED=false
```

可使用以下命令分别生成 JWT 和插件密钥：

```bash
openssl rand -base64 64
```

设置权限：

```bash
chown root:root /etc/xinke/xinke.env
chmod 600 /etc/xinke/xinke.env
```

## 2. 绑定到 systemd 服务

在服务的 `[Service]` 段添加：

```ini
EnvironmentFile=/etc/xinke/xinke.env
```

修改后执行：

```bash
systemctl daemon-reload
systemctl restart xinke
```

## 3. 插件请求头

配置 `LIVE_PLUGIN_REPORT_KEY` 后，插件请求 `/live/viewer/report` 或 `/live/userRoom/report` 时必须携带：

```http
X-Plugin-Key: 与服务器 LIVE_PLUGIN_REPORT_KEY 相同的值
```

未配置 `LIVE_PLUGIN_REPORT_KEY` 时，为兼容旧插件，后端暂不强制校验。正式环境应完成插件配置后再设置该变量。

## 4. 上线检查

- 全站启用 HTTPS，HTTP 只做 301 跳转。
- MySQL、Redis 的 3306/6379 端口不要对公网开放。
- `/swagger-ui/`、`/v3/api-docs/` 和 `/druid/` 保持关闭。
- 修改曾经写入仓库或聊天记录的数据库、Redis 和管理员密码。
- 部署后重新登录，使旧 token 失效，并清理浏览器中旧的 BI token 历史。
