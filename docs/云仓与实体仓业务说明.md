# 云仓与实体仓业务说明

## 1. 设计原则

系统把仓库分为两种形态：

- **实体仓**：公司或外包团队实际操作的仓库。ERP 负责库位、收货、上架、拣货、出库、盘点、报损和库存成本。
- **云仓**：由第三方服务商履约的仓库。ERP 保留自己的库存账，同时接收服务商库存快照并做账实差异核对。

云仓同步不会直接覆盖 ERP 可用库存。ERP 可用库存来自采购入库、销售出库、调拨、盘点和报损流水；云仓实存来自服务商接口或文件。两者的差异用于发现漏单、重复出库、未回传退货和服务商库存错误。

## 2. 上线步骤

1. 备份数据库 `xinkeerp`。
2. 执行 `sql/erp_warehouse_dual_mode_20260716.sql`。
3. 重启后端并重新构建前端。
4. 在“ERP / 仓库管理”中补齐现有仓库资料。
5. 给需要操作云仓同步的角色分配 `erp:inventory:sync` 权限。

## 3. 仓库建档

### 实体仓

选择“实体仓”，维护仓库用途、经营方式、负责人和地址。建议开启库位管理，并按仓内实际结构建立库区和库位，例如 `A-01-01`。

仓库用途：

- 正品仓：可销售商品。
- 退货仓：消费者退回、尚未质检的商品。
- 残次仓：不可按正品销售的商品。
- 中转仓：短期周转或调拨在途落地点。

### 云仓

选择“云仓”，必须维护服务商和外部仓编码。货主编码用于一个服务商管理多个货主时区分库存。同步方式可以先使用手工粘贴，后续接入服务商 API 时不需要改变库存模型。

## 4. 云仓库存对账

进入“ERP / 库存查询”，点击“同步云仓库存”，每行填写：

```text
SKU编码, 可用库存, 锁定库存
DEMO-SKU-001, 120, 5
DEMO-SKU-002, 86, 0
```

系统按 SKU 编码匹配商品档案，并计算：

```text
库存差异 = 云仓实存 - ERP可用库存
```

- 差异为 0：账实一致。
- 差异大于 0：云仓比 ERP 多，检查漏记入库、退货回仓或取消出库。
- 差异小于 0：云仓比 ERP 少，检查漏记出库、丢损或服务商扣减。
- 未建档 SKU：先维护商品和 SKU，不自动创建未知商品。

发现差异后，应通过正确的入库、出库、盘点或报损单调整 ERP 库存，不要直接修改库存余额。

## 5. 关键控制

- 有库存记录的仓库不能删除，只能停用。
- 有库存记录后不能在实体仓和云仓之间切换。
- 停用仓库不能继续出入库。
- 默认禁止负库存；只有明确存在先发货后回传场景时才能开启。
- 云仓不能在 ERP 内维护服务商库位。
- 外部库存快照、同步批次和差异数量会保留同步记录，便于追责和复盘。

## 6. 后续接入建议

服务商 API 接入时，统一调用 `/erp/inventory/cloud/sync`，请求结构与手工同步一致。定时任务只负责获取和转换服务商数据，库存差异、日志和业务规则仍由 ERP 服务处理。
