# 直播追单系统说明书

## 1. 系统用途

直播追单系统用于承接抖音直播互动评论客户，把“有评论、有意向、可跟进”的用户沉淀为销售线索，方便运营分配负责人、记录跟进、标记下单和复盘转化效果。

系统重点解决：

- 评论客户容易遗漏。
- 多个运营跟进责任不清。
- 已下单客户仍被重复追问。
- 直播间 ID 每天变化，业务看不懂。
- 插件异常上报、旧版本上报无法统一控制。
- 直播追单效果缺少数据看板。

## 2. 菜单结构

系统相关页面在“直播运营”目录下：

- 直播追单
- 直播间映射
- 直播追单 BI
- 插件控制

部分工具页面在“系统监控”目录下：

- Token 工具

## 3. 直播追单页面

### 3.1 页面用途

“直播追单”是运营每天使用最多的页面，用来查看插件同步过来的评论客户，并进行负责人分配、跟进记录和订单标记。

### 3.2 筛选条件

常用筛选：

- 日期区间：默认近 7 天。
- 直播间：按直播间名称搜索。
- 客户：按昵称搜索。
- 负责人：按负责人搜索。
- 评论：全部 / 有评论 / 无评论。
- 状态：全部状态 / 新线索 / 跟进中 / 已下单 / 无效等。

导出按钮会按照当前筛选条件导出。如果不填写任何条件，才是导出全部。

### 3.3 列表字段

核心字段：

| 字段 | 含义 |
| --- | --- |
| 客户 | 抖音昵称 |
| 日期 | 客户进入追单池的日期 |
| 来访 | 当天来访只算一次 |
| 直播间 | 通过直播间映射后显示的业务名称 |
| 评论 | 评论条数和最近一条评论 |
| 负责人 | 当前负责跟进的人 |
| 意向 | 有评论自动标记高意向 |
| 状态 | 根据负责人、订单号、时间自动判断 |

### 3.4 状态规则

当前状态规则：

- 有订单号：已下单。
- 有负责人但没有订单号：跟进中。
- 超过一个月仍没有订单号：无效。
- 没有负责人、没有订单号：新线索。

注意：

- “追单前已下单”和“追单后已下单”是两个不同概念。
- 追单前已下单表示客户在进入追单前已经买过，不应该再追问。
- 追单后已下单表示运营跟进后转化成功。

### 3.5 负责人规则

负责人用于锁定当前线索由谁跟进。

规则：

- 负责人为空时，可以填写负责人。
- 负责人有人时，负责人输入框禁用。
- 负责人填写后 24 小时内不能被别人抢走。
- 超过 24 小时仍未下单，负责人可以自动清空，让别人继续跟。
- 如果已经下单，不允许再随意修改负责人。
- 跟进记录和备注会保留，不会因为负责人过期清空。

### 3.6 跟进弹窗

点击“跟进”打开弹窗。

弹窗里主要填写：

- 负责人
- 订单号
- 备注 / 跟进内容

状态和意向不需要人工填写：

- 有负责人就是跟进中。
- 有订单号就是已下单。
- 有评论就是高意向。

已下单必须填写订单号。

### 3.7 详情页

点击“详情”可以查看：

- 客户基础信息
- 抖音主页复制按钮
- 评论记录
- 来访记录
- 跟进记录

评论记录显示真实评论明细。

来访记录按日期聚合：

- 同一天来访算一次。
- 评论条数直接按评论明细表统计。
- 最近评论显示当天最新一条评论。

## 4. 直播间映射页面

### 4.1 页面用途

抖音直播间 ID 会变化，运营不容易识别。直播间映射页面用于把插件上报的直播间 ID 映射成业务名称。

例如：

- `7659213161007565619` -> `希沃学习机旗舰店`

### 4.2 使用方式

1. 打开“直播运营 / 直播间映射”。
2. 点击“新增”。
3. 填写直播间名称。
4. 填写直播间 ID。
5. 状态选择启用。
6. 保存。

追单列表会优先展示映射后的直播间名称。

### 4.3 注意事项

- 如果直播间 ID 每天变化，可以保留历史 ID 映射。
- 如果同一个店铺有多个直播间 ID，都可以映射到同一个直播间名称。
- 不建议删除历史映射，否则旧数据可能只能显示 ID。

## 5. 插件控制页面

### 5.1 页面用途

插件控制页面用于统一管理插件是否允许上报，以及控制旧版本插件、异常电脑。

### 5.2 采集总开关

采集总开关关闭后：

- 插件下一次上报会收到停止响应。
- 插件会自动停止监听。
- 本地待上报队列保留。

适用场景：

- 插件异常上报。
- 系统维护。
- 临时冻结采集。

### 5.3 版本策略

页面可以配置：

- 最低允许版本
- 最新版本

如果插件版本低于最低允许版本，后端会拒绝上报，并让插件自动停止。

当前插件版本显示在插件弹窗顶部，例如：`v4.0.8`。

### 5.4 插件电脑列表

每台电脑首次运行插件会生成唯一编号。

页面可以看到：

- 电脑名称
- 电脑编号
- 状态
- 插件版本
- 直播间
- 上报次数
- 异常次数
- 最后上报时间
- 最后成功时间

可以单独禁用某台电脑。

禁用后，该电脑下一次上报会收到 `46003`，插件自动停止，并显示“电脑已禁用”。

### 5.5 停止响应码

| 编码 | 含义 | 插件动作 |
| --- | --- | --- |
| 46001 | 采集总开关关闭 | 停止插件 |
| 46002 | 插件版本过低 | 停止插件 |
| 46003 | 当前电脑被禁用 | 停止插件 |

## 6. 直播追单 BI 页面

### 6.1 页面用途

直播追单 BI 用于复盘直播间线索质量、负责人跟进效果和订单转化。

适合运营主管每天查看。

### 6.2 核心指标

看板常见指标：

- 覆盖店铺
- 去重观众
- 高意向客户
- 未分配高意向
- 待跟进客户
- 追后成交

### 6.3 常用分析

#### 线索趋势

查看每天观众、评论客户、追后成交变化。

用途：

- 判断直播间流量是否稳定。
- 判断评论客户是否上升。
- 判断追单转化是否有效。

#### 追单漏斗

从进入追单池到追后成交。

用途：

- 看哪一步流失严重。
- 发现未分配、未跟进、未成交的问题。

#### 评论热力

按小时统计评论密度。

用途：

- 找到客户最活跃的时间段。
- 辅助安排运营值班。

#### 成交信号词云

从评论中识别购买信号。

用途：

- 判断客户关注点。
- 识别更接近成交的评论。

#### 负责人绩效

查看负责人分配、跟进和成交情况。

用途：

- 发现跟进效率高的人。
- 发现未及时处理的负责人。

#### 运营缺口

系统自动提示风险，例如：

- 高意向客户未分配。
- 负责人跟进覆盖不足。
- 追后成交率偏低。
- 直播间名称未维护。

## 7. Token 工具页面

### 7.1 页面用途

Token 工具用于生成当前登录用户的 token，方便把直播 BI 页面嵌入第三方 BI 工具。

### 7.2 注意事项

- token 相当于登录凭证，不要随意发给无关人员。
- token 过期后，嵌入页面会提示登录过期或数据加载失败。
- 如果 BI 工具里写死 token，需要定期手动更换。

## 8. 数据规则

### 8.1 去重规则

评论去重：

- 同一天
- 同昵称
- 同评论内容

满足以上三项相同，视为重复评论，不重复写入。

来访去重：

- 同一天来访只算一次。

用户建档：

- 优先按 `secUid` 建档。
- 插件只有评论匹配到在线观众资料时，才上传用户资料。

### 8.2 高意向规则

有评论的客户自动标记为高意向。

原因：

- 评论代表主动互动。
- 追单系统要优先处理有表达的客户。

### 8.3 无效规则

超过一个月仍没有订单号，可以视为无效。

无效不代表删除数据，只是降低跟进优先级。

## 9. 日常工作流程

### 9.1 直播前

1. 打开抖音主播后台。
2. 确认插件显示“监听中”。
3. 打开系统“直播间映射”，确认当前直播间名称维护正确。
4. 打开“插件控制”，确认采集总开关开启。

### 9.2 直播中

1. 插件自动采集评论。
2. 运营可以定时点击“立即上报”。
3. 在“直播追单”查看新增高意向客户。
4. 给客户分配负责人。
5. 负责人及时跟进并填写记录。

### 9.3 直播后

1. 查看“直播追单”未分配高意向客户。
2. 补充负责人和跟进记录。
3. 已成交客户填写订单号。
4. 打开“直播追单 BI”复盘转化效果。

## 10. 常见问题

### 10.1 插件显示成功，但系统没有线索

可能原因：

- 该评论没有匹配到在线观众资料。
- 评论等待 1 分钟后被插件丢弃。
- 后端插件控制关闭。
- 当前电脑被禁用。
- 插件版本过低。

处理方式：

1. 打开插件日志。
2. 展开“采集评论”日志。
3. 看是否显示“已匹配”。
4. 如果显示“等待匹配”或“已丢弃”，系统不会生成线索。
5. 如果显示上报失败，到“插件控制”检查开关、版本和电脑状态。

### 10.2 详情里评论数和评论记录不一致

已修复为按评论明细表统计。

如果历史数据异常，可以执行：

`sql/douyin_comment_count_repair_20260706.sql`

### 10.3 直播间显示的是 ID

说明没有维护直播间映射。

处理方式：

1. 打开“直播间映射”。
2. 新增该直播间 ID 和名称。
3. 返回追单列表刷新。

### 10.4 负责人不能修改

可能原因：

- 已经有人负责该客户。
- 负责人锁定期未过。
- 客户已经下单。

这是为了避免多人抢同一个客户。

### 10.5 已下单用户还出现在追单池

需要区分：

- 追单前已下单：进入追单前已经购买，不需要追问。
- 追单后已下单：运营跟进后成交，是有效转化。

系统需要分别标记，不能混成一个状态。

## 11. 管理建议

- 每天直播结束后检查未分配高意向客户。
- 负责人填写后要及时补跟进记录。
- 已下单必须填写订单号。
- 定期查看插件控制里的异常电脑。
- 插件升级后，把最低允许版本调到新版本，防止旧插件继续上传。
- 不建议删除历史直播间映射和历史追单数据。
