# 鑫科 ERP 财务模块使用与设计说明书

## 1. 文档目的

本说明书用于财务、出纳、采购负责人、运营负责人和系统管理员培训，说明每个财务页面解决什么问题、按钮如何使用、单据如何流转，以及系统为什么采用当前设计。

本系统采用“业务单据 -> 应收应付 -> 真实资金流水 -> 核销 -> 会计凭证 -> 账簿 -> 月结”的处理链路。利润报表用于经营分析，资金流水和会计账簿用于财务核算，两者不能混为一张表。

## 2. 角色与职责

| 角色 | 主要职责 | 不建议授予的权限 |
|---|---|---|
| 业务录入员 | 录入平台结算、费用、供应商发票和付款申请 | 付款执行、反关账 |
| 财务审核员 | 审核结算、费用、发票、付款申请和凭证 | 银行付款执行 |
| 出纳 | 登记银行流水、执行已审批付款、核销应收应付 | 修改业务原始金额、反关账 |
| 财务主管 | 对账差异处理、月结、特殊例外审批 | 无必要时不直接录入原始单据 |
| 系统管理员 | 账号、权限和基础资料维护 | 不代替财务完成日常审核 |
| 只读人员 | 查看报表、应收应付和账簿 | 所有新增、审核、付款和关账权限 |

关键原则：录入、审批和付款执行应尽量由不同人员完成。即使小公司暂时由同一人兼任，也应保留不同按钮和操作日志。

## 3. 总体业务流程

### 3.1 平台销售收款

1. 在“平台结算”录入平台账单中的收入、退款和各项扣费。
2. 财务确认结算单，系统自动生成应收账款。
3. 在“资金流水”登记平台实际到账。
4. 在“应收账款”选择该笔收款流水进行核销。
5. 在“对账差异”检查账单应收和实际到账差异。
6. 生成并检查凭证，过账后进入账簿。

### 3.2 采购与供应商付款

1. ERP 完成采购订单和采购入库。
2. 在“应付与付款 / 供应商发票”登记供应商发票。
3. 系统执行采购订单、入库数量、发票数量及价格的三单匹配。
4. 匹配通过后确认应付；有差异时必须填写例外放行原因。
5. 在应付单上发起付款申请。
6. 审批人选择通过或驳回；驳回必须填写原因。
7. 出纳对已审批申请执行付款，必须填写银行流水号。
8. 系统自动生成支出流水、付款执行记录、应付核销记录和付款凭证。

### 3.3 日常费用

1. 在“费用管理”登记推广费、物流费、服务费等费用。
2. 草稿阶段可修改或作废。
3. 审核后自动生成应付，原费用单不再允许直接修改。
4. 需要会计核算时生成凭证并过账。

### 3.4 月结

1. 处理草稿结算单、草稿费用单和未入账流水。
2. 完成应收应付核销和平台对账。
3. 检查并过账所有会计凭证。
4. 在“账簿中心”确认试算平衡。
5. 在“月结管理”运行检查，全部通过后关账。
6. 关账后禁止修改该期间数据；确需更正时由财务主管填写原因并反关账。

## 4. 页面使用说明

## 4.1 财务运营中心

### 解决的问题

统一展示待收款、待付款、本月资金流入流出、财务待办、最近财务动作和六个月资金趋势，避免用户在多个菜单之间盲目查找。

### 操作说明

| 区域或按钮 | 用法 | 注意事项 |
|---|---|---|
| 刷新数据 | 重新读取最新财务汇总 | 不会重新计算历史 ETL 利润数据 |
| 待收款卡片 | 进入应收账款 | 金额取未结清应收余额 |
| 待付款卡片 | 进入应付与付款 | 金额取未结清应付余额 |
| 本月流入/流出 | 进入资金流水 | 只统计已入账流水，不含草稿 |
| 本期待办 | 进入对应处理页面 | 数量为零表示当前没有该类待办 |
| 财务处理顺序 | 按标准流程进入页面 | 不建议跳过审核、核销或凭证过账 |
| 业务模块 | 直接进入具体财务页面 | 所有卡片均可点击 |
| 经营利润 | 查看运营、渠道、店铺日月利润 | 依赖利润 ETL 数据，不等同于总账利润表 |

### 仪表加载失败处理

接口为 `GET /finance/dashboard/overview`。如果页面显示“系统处理失败”而日志提示该地址是静态资源或接口不存在，说明正在运行的后端包未包含最新的 `FinanceDashboardController`，需要重新构建并重启后端。源代码已将此类情况改为明确的 404 提示。

## 4.2 财务基础设置

### 解决的问题

集中查看会计期间、科目、费用类型、平台资金账户和银行账户，确保业务人员使用统一基础资料。

### 操作说明

| 按钮 | 用法 | 设计原因 |
|---|---|---|
| 刷新基础资料 | 重新载入所有基础资料 | 基础资料变更频率低，不需要自动频繁请求 |
| 页签切换 | 查看不同类型资料 | 当前页面是受控资料查看页，不提供普通业务人员随意修改 |

银行账户、会计科目和会计期间会影响付款、凭证和月结。当前版本将其作为受控资料，避免误删科目或银行账户导致历史数据失去关联。基础资料变更应由系统管理员和财务主管共同确认。

## 4.3 平台结算

### 解决的问题

把抖音、天猫、京东等平台账单中的销售收入、退款、佣金、广告费、支付手续费、服务费和运费转成标准应收单据。

### 按钮说明

| 按钮 | 可用状态 | 结果 |
|---|---|---|
| 新增结算单 | 任意 | 新建草稿，系统自动计算平台应收 |
| 修改 | 草稿 | 修正平台、店铺、期间和金额 |
| 作废 | 草稿 | 单据标记为已作废，不生成应收或凭证 |
| 确认并生成应收 | 草稿 | 锁定结算金额并生成应收账款 |
| 生成凭证 | 已确认且未生成凭证 | 生成待检查的会计凭证草稿 |
| 搜索/重置/刷新 | 任意 | 只改变查询结果，不改变业务数据 |

### 注意事项

- 收入、退款和各项扣费不能为负数。
- 账单开始日期不能晚于结束日期。
- 平台应收 = 收入 - 退款 - 佣金 - 支付手续费 - 广告费 - 服务费 - 运费 - 其他扣费。
- 平台应收不能小于零；出现负数时应核对账单或拆分跨期调整。
- 确认后不能直接修改，应通过差异处理或后续更正流程保留审计轨迹。

## 4.4 应收账款

### 解决的问题

回答“平台或客户还欠公司多少钱、哪笔钱已经到账、到账流水核销到哪张应收单”。

### 按钮说明

| 按钮 | 用法 | 约束 |
|---|---|---|
| 选择收款核销 | 选择一笔已入账的收入流水并输入核销金额 | 金额不能超过应收余额或流水可用余额 |
| 确定核销 | 写入核销记录并同步更新应收和流水状态 | 同一余额发生并发变化时系统会回滚并要求刷新 |

不得使用手工填写的虚假流水号核销。必须先在资金流水登记实际到账。

## 4.5 应付与付款中心

页面包含“应付账款、供应商发票、付款申请”三个页签。

### 应付账款按钮

| 按钮 | 用法 | 约束 |
|---|---|---|
| 申请付款 | 以应付可申请余额生成付款申请 | 已提交、已审批但未支付的金额会占用可申请余额 |
| 手工核销 | 用已入账的支出流水核销应付 | 仅用于系统外已经付款但未走系统付款申请的场景 |

优先使用“申请付款 -> 审批 -> 执行付款”。“手工核销”是补录工具，不应代替正常付款审批。

### 供应商发票按钮

| 按钮 | 用法 | 结果 |
|---|---|---|
| 新增供应商发票 | 选择采购订单并登记发票数量、单价和税率 | 自动执行采购、入库、发票三单匹配 |
| 匹配明细 | 查看每个 SKU 的订单、入库、开票数量和差异 | 只读 |
| 确认应付 | 匹配通过后确认 | 自动生成应付和供应商发票凭证 |
| 确认例外放行 | 差异发票填写原因后确认 | 原差异和放行原因永久保留 |

### 付款申请按钮

| 按钮 | 可用状态 | 结果 |
|---|---|---|
| 审批 | 待审批 | 付款申请进入待付款 |
| 驳回 | 待审批 | 释放占用的可申请余额，并在备注中记录驳回原因 |
| 执行付款 | 已审批或部分付款 | 生成银行支出流水、付款执行记录、应付核销和付款凭证 |

执行付款必须选择有效银行账户、填写付款金额、付款时间和银行流水号。银行流水号用于后续银行对账，不应填写“无”或随机内容。

## 4.6 资金出入账中心

### 解决的问题

统一管理银行账户和平台账户的真实资金流入流出，区分业务金额、手续费和账户净额。

### 按钮说明

| 按钮 | 可用状态 | 结果 |
|---|---|---|
| 新增流水 | 任意 | 可保存草稿或立即入账 |
| 入账 | 草稿 | 锁定流水，并可同时生成会计凭证 |
| 作废 | 草稿 | 草稿作废，不影响资金和账簿 |
| 冲销 | 已入账且未核销 | 生成一笔反向流水和冲销凭证，原流水保留 |
| 刷新/查询/重置 | 任意 | 只刷新或筛选列表 |

已经核销的流水不能直接冲销，因为它与应收应付存在关联。应先由财务主管处理核销关系，再执行冲销。已入账流水不能删除，防止资金记录无痕消失。

## 4.7 费用管理

### 解决的问题

归集推广、物流、平台服务、办公和其他费用，并按店铺、日期、订单或 SKU 分摊。

### 按钮说明

| 按钮 | 可用状态 | 结果 |
|---|---|---|
| 新增费用 | 任意 | 新建费用草稿 |
| 修改 | 草稿 | 修正费用类型、金额、税额和分摊维度 |
| 作废 | 草稿 | 不生成应付和凭证 |
| 审核并生成应付 | 草稿 | 锁定费用并生成应付 |
| 生成凭证 | 已审核且未生成凭证 | 生成费用凭证草稿 |

费用金额必须大于零，税额不能小于零。费用审核前应检查供应商、订单、SKU 和分摊维度，否则利润分析会落入错误维度。

## 4.8 对账差异

### 解决的问题

比较平台结算应收与实际收款，找出缺少流水和金额不一致的问题。

### 按钮说明

| 按钮 | 用法 | 结果 |
|---|---|---|
| 运行对账 | 选择期间和店铺后执行 | 生成对账任务及差异明细 |
| 填写处理结果 | 对开放差异填写核对结论 | 差异转为已处理并保留处理人、时间和说明 |

“填写处理结果”只表示差异已经人工核实，不会凭空生成流水或修改余额。需要调整金额时，应另行补录流水或更正业务单据。

## 4.9 凭证管理

### 解决的问题

检查业务自动生成的会计凭证，并把借贷分录正式写入账簿。

### 按钮说明

| 按钮 | 用法 | 约束 |
|---|---|---|
| 分录 | 查看借方、贷方、科目和摘要 | 只读检查 |
| 检查并过账 | 对草稿凭证执行借贷检查并写入正式账簿 | 无分录或借贷不平衡时禁止过账 |

凭证过账前必须检查科目和摘要。过账后不能直接修改；错误凭证应使用红字或冲销凭证更正，而不是删除历史凭证。

## 4.10 账簿中心

### 解决的问题

查看正式明细账和试算平衡。只有已经过账的凭证进入本页面。

### 操作说明

| 操作 | 用法 |
|---|---|
| 明细账页签 | 按期间、凭证号、科目和来源单号查询正式分录 |
| 试算平衡页签 | 查看各科目期初、本期发生和期末借贷余额 |
| 查询/重置 | 应用或清空筛选条件 |

试算不平衡时禁止月结，应先检查是否存在历史导入数据或未正确过账的凭证。

## 4.11 月结管理

### 解决的问题

冻结已完成期间，防止报表出具后业务人员继续修改历史数据。

### 按钮说明

| 按钮 | 可用状态 | 结果 |
|---|---|---|
| 月结检查 | 任意开放期间 | 统计草稿、未核销、未过账和未结余额 |
| 重新检查 | 未关账 | 更新检查结果 |
| 确认关账 | 检查通过 | 关闭会计期间，阻止该期间继续修改 |
| 反关账 | 已关账 | 填写审批原因后重新开放期间 |

反关账属于高风险权限，只应授予财务主管。原因至少五个字，实际管理中还应填写审批单号或负责人。

## 5. 状态说明

### 5.1 通用草稿状态

| 状态 | 含义 |
|---|---|
| draft | 草稿，可修改或作废 |
| approved / confirmed | 已审核或已确认，不能直接修改 |
| posted | 已入账或已过账，进入正式账簿或资金记录 |
| partial | 部分核销或部分付款 |
| settled / paid / matched | 已结清、已付清或已匹配 |
| voided | 草稿已作废，不参与后续处理 |
| reversed | 已通过反向单据冲销 |
| rejected | 审批驳回，可按原因重新发起业务 |

## 6. 关键设计原因

### 6.1 为什么确认后不能直接修改

确认后的单据已经生成应收、应付、凭证或核销关系。直接修改原金额会造成上下游数据不一致。系统只允许修改草稿，确认后使用冲销、更正或差异处理。

### 6.2 为什么核销必须选择真实流水

应收应付表示“应该收付”，资金流水表示“实际收付”。只有把两者关联，系统才能准确回答哪些钱已经到账、哪些供应商已经付款。

### 6.3 为什么付款执行必须有银行流水号

付款审批只代表同意付款，银行流水号才是资金真正付出的证据。它也是银行对账、重复付款排查和审计抽查的重要依据。

### 6.4 为什么付款申请必须可以驳回

收款账户、金额、付款日期或附件错误时，应释放占用金额并保留驳回原因，不能让错误申请永久停留在待审批列表。

### 6.5 为什么使用冲销而不是删除

删除会让历史记录消失，无法解释账户余额变化。冲销同时保留原流水和反向流水，审计链完整。

### 6.6 为什么增加并发保护

双击按钮或多人同时操作可能导致重复核销、重复付款或重复生成下游单据。系统使用状态条件和旧余额校验；检测到数据已变化时整笔事务回滚，并要求用户刷新后重试。

## 7. 日常操作清单

### 每日

1. 登记或导入当日银行、平台资金流水。
2. 处理待审批付款申请和已审批待付款申请。
3. 核销应收和应付。
4. 处理平台结算、供应商发票和费用草稿。
5. 检查对账差异和异常流水。
6. 检查自动生成凭证，确认无误后过账。

### 每月

1. 确认当月所有平台账单已录入。
2. 确认采购入库和供应商发票完整。
3. 核对银行、平台账户余额和未达账项。
4. 检查应收应付账龄。
5. 处理所有开放对账差异。
6. 过账全部凭证并检查试算平衡。
7. 运行月结检查并关账。

## 8. 常见问题

### 按钮看得见但点击后提示无权限

管理员需要执行 `sql/finance_operation_control_upgrade_20260719.sql` 并给角色授予相应按钮权限。新增权限包括结算草稿修改、费用草稿修改、流水草稿作废和对账差异处理。

### 找不到账簿中心

执行 `sql/finance_accounting_ledger_upgrade_20260719.sql`，重新登录以刷新动态菜单和权限。

### 财务仪表显示系统处理失败

确认后端已经重新构建和重启，并检查 `/finance/dashboard/overview` 是否返回 JSON。旧运行包没有该控制器时会出现接口不存在。

### 手工核销列表没有可选流水

只有方向正确、已入账且还有可用余额的流水会显示。应收使用收入流水，应付使用支出流水。

### 付款申请无法再次发起

已提交、已审批或部分付款的申请会占用应付可申请余额。先处理原申请；驳回后占用金额会自动释放。

## 9. 上线前检查

1. 执行财务框架、应付中心、账簿中心和操作控制 SQL 升级脚本。
