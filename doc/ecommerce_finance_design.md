# 电商行业财务管理系统设计

## 1. 设计定位

电商财务系统的目标不是只算利润，而是把平台订单、退款、广告、运费、采购、库存成本、平台结算、银行收款、应收应付、费用和凭证串成一条可追溯链路。

系统分两套口径：

- 经营利润口径：面向老板和运营，强调店铺、商品、渠道、日期维度的实时毛利和 ROI。
- 财务会计口径：面向财务关账，强调收入确认、成本结转、费用归集、往来核销、凭证和月结。

当前系统已有 `dws_finance_day_profit`，它是经营利润表的雏形。下一步要补财务主数据、结算、对账、往来、费用、凭证和月结。

## 2. 核心业务闭环

### 2.1 销售收入闭环

订单支付：

- ERP 保存平台原始订单和清洗后的订单明细。
- 经营口径可先按支付金额计入 GMV。
- 财务口径要区分已支付、已发货、已完成、已退款。

平台结算：

- 平台账单导入后形成平台结算单。
- 结算单拆分收入、退款、平台佣金、支付手续费、广告费、达人佣金、运费险等。
- 平台结算单与订单、退款、银行到账做三方对账。

收款核销：

- 银行或第三方支付流水导入。
- 按平台、店铺、结算周期匹配平台结算单。
- 形成应收核销，差异进入对账差异表。

### 2.2 成本闭环

采购入库：

- 采购单确认后形成预计成本。
- 入库后形成库存成本。
- 成本方法默认先用标准成本，后续可升级移动加权平均。

销售出库：

- 发货或订单完成时结转主营业务成本。
- 售后退货入库需要冲回成本或进入残次库存。

成本调整：

- 采购价差、盘盈盘亏、残次处理通过成本调整单处理。
- 所有调整必须有来源单据和审批记录。

### 2.3 费用闭环

费用类型：

- 平台佣金
- 支付手续费
- 推广费用
- 达人佣金
- 运费
- 仓储费
- 包装耗材
- 售后补偿
- 退款损失
- 管理费用

费用归集：

- 可归属到店铺、商品、SKU、订单、月份。
- 能归到订单的费用进入订单利润。
- 不能归到订单的费用进入店铺/月度费用分摊。

### 2.4 往来闭环

应收：

- 平台待结算款
- 客户应收款
- 其他应收款

应付：

- 供应商采购应付
- 物流应付
- 达人/服务商应付
- 平台费用应付

核销：

- 收款核销应收。
- 付款核销应付。
- 差异需生成调整记录，不允许直接删除。

### 2.5 凭证和月结

凭证来源：

- 平台结算单
- 采购入库/采购发票
- 销售成本结转
- 费用单
- 收付款流水
- 售后退款
- 成本调整

月结动作：

1. 检查订单、退款、结算、广告、物流、采购数据是否导入完整。
2. 检查平台结算与银行到账差异。
3. 计算收入、成本、费用、毛利、净利。
4. 生成或锁定凭证。
5. 锁定会计期间，禁止普通用户修改已结账数据。

## 4. 核心表设计

### 4.1 财务主数据

- `fin_account_period`：会计期间
- `fin_account_subject`：会计科目
- `fin_platform_account`：平台资金账户

### 4.2 结算与对账

- `fin_platform_settlement`：平台结算单
- `fin_platform_settlement_item`：平台结算明细

- `fin_reconciliation_diff`：对账差异

### 4.3 往来与收付款

- `fin_receivable_bill`：应收单
- `fin_payable_bill`：应付单
- `fin_cash_flow`：收付款流水

### 4.4 成本与费用

- `fin_expense_bill`：费用单
- `fin_cost_ledger`：成本台账

### 4.5 凭证与月结

- `fin_voucher`：凭证头
- `fin_voucher_entry`：凭证分录
- `fin_period_close`：期间结账

## 5. 关键计算口径

经营利润：

```text
经营利润 = 支付金额
         - 商品成本
         - 平台扣点
         - 支付手续费
         - 推广费用
         - 达人佣金
         - 运费
         - 仓储包装费
         - 售后成本
```

平台应收：

```text
平台应收 = 销售收入
         - 退款金额
         - 平台佣金
         - 支付手续费
         - 平台代扣费用
```

应付供应商：

```text
供应商应付 = 入库金额 + 税额 - 退货金额 - 已付款金额
```

ROI：

```text
ROI = 净成交额 / 推广费用
```

## 6. 对账规则

三方对账关系：

- 订单明细：业务事实
- 平台结算：平台确认金额
- 银行流水：实际到账

差异类型：

- 平台漏结
- 重复结算
- 退款跨期
- 手续费差异
- 广告扣费差异
- 银行到账差异
- 人工调整

差异状态：

- `open` 待处理
- `confirmed` 已确认
- `adjusted` 已调整
- `ignored` 已忽略

## 8. MVP 开发顺序

第一阶段：

- 财务主数据：期间、科目、费用类型、银行账户、平台账户。
- 平台结算单：列表、导入、明细。
- 收付款流水：列表、导入。

第二阶段：

- 结算单与银行流水对账。
- 应收应付单生成。
- 费用单录入和费用归集。

第三阶段：

- 成本台账。
- 凭证生成。
- 月结锁账。

第四阶段：

- 财务报表：利润表、现金流、应收应付账龄、店铺利润、商品利润。
- 异常预警：未结算订单、未核销应收、负毛利订单、跨期退款。

## 9. 与现有系统衔接

现有表：

- `dwd_order_item` 提供订单利润明细。
- `dws_finance_day_profit` 提供经营利润日报/月报。
- `erp_settlement_bill` 是 ERP 层的简化结算账单，可逐步迁移到 `fin_platform_settlement`。

新增财务表用 `fin_` 前缀，表示财务会计域。ERP 表负责业务事实，`fin_` 表负责财务确认、对账、核销和凭证。
