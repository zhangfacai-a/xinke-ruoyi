# 电商 ERP 核心设计

## 1. 设计目标

本项目的 ERP 不是简单的订单列表，而是面向电商经营的数据操作系统。ERP 负责沉淀业务事实，BI 和财务核算只消费 ERP/ETL 产出的可信数据。

核心目标：

- 打通商品、SKU、店铺、仓库、供应商、库存、订单、售后、结算、费用。
- 每一笔销售、退款、采购、库存变动都能追溯来源单据。
- 运营系统和 BI 指标共用同一套明细事实，避免口径不一致。
- 前期采用 RuoYi CRUD 快速落地，后期逐步补工作流、审核、导入和平台对接。

## 2. ERP 模块地图

### 2.1 基础资料

- 店铺档案：平台、店铺编码、负责人、状态。
- 仓库档案：自营仓、三方仓、退货仓、默认发货仓。
- 供应商档案：结算方式、账期、联系人、状态。
- 客户档案：B2B 客户、分销商、平台买家聚合档案。
- 费用类型：平台扣点、推广费、运费、包装费、补偿费、仓储费。

### 2.2 商品中心

- SPU 商品：货号、类目、品牌、生命周期状态。
- SKU：规格、条码、成本价、建议售价、重量体积。
- SKU 组合/BOM：套装商品、赠品包、组合拆分。
- 渠道映射：平台 SKU、店铺 SKU 与内部 SKU 的对应关系。
- 成本维护：标准成本、最新采购成本、移动加权成本。

### 2.3 库存仓储

- 库存余额：仓库 + SKU 当前可用、锁定、在途、残次、预警库存。
- 库存流水：入库、出库、调拨、盘点、锁定、解锁、退款入库。
- 调拨单：仓库之间库存转移。
- 盘点单：盘盈盘亏。
- 预占库存：订单支付后锁定，发货后扣减。

### 2.4 采购供应

- 采购订单：供应商、预计到货、采购状态。
- 采购明细：SKU、数量、采购价、税率。
- 采购入库：到货验收后生成库存流水。
- 采购退货：残次或超收退回供应商。
- 应付账款：按入库或发票形成应付。

### 2.5 销售订单

- 平台原始订单：ODS 原样保存平台 JSON。
- ERP 销售订单：统一订单头、买家、店铺、渠道状态。
- 订单明细：内部 SKU 拆分、数量、支付金额、优惠。
- 订单状态流转：待付款、已付款、待发货、已发货、完成、关闭。
- 发货单：承运商、运单号、发货仓、运费。

### 2.6 售后

- 退款单：仅退款、退货退款、补偿。
- 退货入库：售后完成后回写库存。
- 售后成本：退款、补偿、逆向运费、残损成本。
- 售后原因：质量、错发、未收到、七天无理由等。

### 2.7 结算财务

- 平台结算账单：收入、退款、平台佣金、支付手续费。
- 推广费用：平台广告、达人佣金、站外投放。
- 利润核算：支付金额 - 商品成本 - 平台扣点 - 推广费 - 运费 - 售后成本。
- 应收应付：供应商应付、平台应收、客户应收。
- 日/月利润报表：店铺、商品、渠道维度。

### 2.8 导入与 ETL

- 导入批次：订单、退款、结算、广告、采购。
- ODS：平台原始数据。
- DWD：清洗后的订单、退款、费用明细。
- DWS：店铺日销售、商品日销售、财务日利润。
- ADS：BI 展示层。

## 3. 第一批落地范围

当前先落基础资料和库存底座：

- `erp_shop` 店铺档案
- `erp_warehouse` 仓库档案
- `erp_supplier` 供应商档案
- `erp_inventory_balance` SKU 库存余额
- `erp_inventory_movement` 库存流水

第一批页面：

- ERP / 店铺管理
- ERP / 仓库管理
- ERP / 供应商管理
- ERP / 库存查询

第一批接口：

- `GET /erp/shop/list`
- `GET /erp/warehouse/list`
- `GET /erp/supplier/list`
- `GET /erp/inventory/list`

## 4. 单据状态约定

基础状态：

- `0` 正常/启用
- `1` 停用/关闭

业务单据状态：

- `draft` 草稿
- `auditing` 审核中
- `approved` 已审核
- `processing` 处理中
- `done` 已完成
- `closed` 已关闭
- `void` 已作废

库存流水方向：

- `IN` 入库
- `OUT` 出库
- `LOCK` 锁定
- `UNLOCK` 解锁
- `ADJUST` 调整
- `TRANSFER` 调拨

## 5. 后续开发顺序

1. 基础资料与库存余额。
2. 采购订单、采购入库、库存流水。
3. 销售订单、订单 SKU 拆分、库存预占。
4. 售后退款、退货入库、售后成本。
5. 平台结算、广告费用、财务利润核算。
6. 导入模板、批次追踪、异常数据修复。
7. BI 指标口径固化和 ADS 看板。
